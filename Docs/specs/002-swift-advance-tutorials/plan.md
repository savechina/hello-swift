## Phase 0: Research (Phase 3 Decisions)

### Decision 1: Vapor vs Hummingbird Web Framework

**Decision**: Vapor  
**Rationale**: Vapor is the most popular Swift web framework (largest community, extensive documentation, Fluent ORM integration). Matches hello-rust axum coverage.  
**Alternatives Considered**: Hummingbird (lighter weight, SwiftNIO-native) - rejected due to smaller ecosystem and fewer learning resources.

### Decision 2: GRDB vs SQLite.swift vs Core Data

**Decision**: GRDB  
**Rationale**: GRDB supports macOS and Linux cross-platform (unlike Core Data/SwiftData). Type-safe Record protocol, QueryInterface DSL, Associations support. Matches hello-rust diesel/sqlx coverage.  
**Alternatives Considered**: SQLite.swift (raw SQL, no ORM features) - rejected; Core Data (macOS only, legacy API) - rejected.

### Decision 3: Actors Chapter Depth

**Decision**: 2 chapters (Basics + Sendable Deep)  
**Rationale**: Actors are Swift 6 concurrency core feature. Single chapter insufficient to cover actor isolation, Sendable protocol, compiler checks. Split allows progressive learning.  
**Alternatives Considered**: 1 chapter (combined) - rejected as too dense; 3+ chapters - rejected as overkill for tutorial scope.

### Decision 4: Swift-Specific Features Priority

**Decision**: All 7 features in single phase  
**Rationale**: 
- **P0 (Critical)**: Property Wrappers (SwiftUI uses), ARC (memory safety), Opaque Types (API design)
- **P1 (Important)**: Unsafe Pointers (C interop), Macros (SwiftData uses)
- **P2 (Advanced)**: Result Builders (DSL), Mirror (reflection)
Grouping allows efficient coverage without excessive fragmentation.  
**Alternatives Considered**: Split into Phase 4 - rejected as features are interrelated and better learned together.

### Decision 5: Chapter Template Adaptation

**Decision**: Use hello-rust 15-section template (expanded from constitution's 12-section)  
**Rationale**: Constitution III specifies 12-section, but hello-rust template adds "Swift vs Rust/Python 对比表" and expanded 动手练习 levels. This expansion approved for advance content cross-language learning.  
**Alternatives Considered**: Strict 12-section - rejected as comparison tables improve Rust→Swift migration experience.
